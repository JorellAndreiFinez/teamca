Controllers

Controllers are request handlers that extract HTTP request data and delegate business logic to services.

Auth Controller

Section titled “Auth Controller”

Manages user authentication flows.

Location: src/controllers/authController.ts

Key Endpoints:

EndpointMethodPurpose
/auth/loginPOSTAuthenticate user, return JWT
/auth/logoutPOSTInvalidate session
/auth/refreshPOSTGet new JWT using refresh token

User Controller

Section titled “User Controller”

Manages user CRUD operations and user management.

Location: src/controllers/userController.ts

Key Endpoints:

EndpointMethodPurpose
/usersGETList all users (paginated)
/users/:idGETGet user details
/usersPOSTCreate new user (Admin+)
/users/:idPATCHUpdate user info
/users/:id/rolePATCHUpdate user role (Superadmin only)

Task Controller

Section titled “Task Controller”

Handles task creation, updates, and retrieval.

Location: src/controllers/taskController.ts

Key Endpoints:

EndpointMethodPurpose
/tasksGETList tasks (filtered by permissions)
/tasks/:idGETGet task details
/tasksPOSTCreate new task
/tasks/:idPATCHUpdate task
/tasks/:id/statusPATCHChange task status
/tasks/:id/commentsPOSTAdd comment to task
/tasks/:id/feedbackPOSTAdd feedback to task

Request/Response Examples:

Access Control:

  • Create: Any authenticated user
  • Update: Task creator or Admin
  • View: Own tasks or tasks in same department
